# Чат с валидатором + финальный экран флоу

**Источник:** Figma 2537:325679 (чат) · 2542:437478 (чат свёрнут) · 2542:437931
(финальный экран — «Отвалидирован» + баннер «юрисдикция»).
**Цель:** завершить флоу создания кооператива — работа с валидатором через чат и
финальный экран «всё готово».

## Reuse-аудит — новых базовых компонентов нет
| Часть | Решение |
|---|---|
| Инфо документа/пайщики/документы | новый общий **`CooperativeInfo`** (QuestionCard + Badge) — переиспользуется на экране голосования и в чате |
| Чат | DS `ChatBubble` + Button + input |
| Баннеры | DS `Banner` (caution → info) |
| Финальный экран | `AccountsScreen` + `useEnsureFinal` |

## Фазы
### [x] Фаза 1. Состояние
`ValidationStage` += `validated`. `completeValidation()` (крестик в чате) ·
`seedFinal()` (полный сид: совет/счета/распределение/подсчёт/валидация=validated +
podschetVoteDone). Хук `useEnsureFinal()`.

### [x] Фаза 2. Экраны
- `cooperative-info.tsx` — общий блок (Информация о кооперативе + Пайщики +
  Документы); пайщики динамические из flow. Слот `afterPaishiki` (для «Вопросов
  голосования» на экране голосования).
- `validator-chat-screen.tsx` (шаг 28) — «Уставные документы»: CooperativeInfo +
  чат-панель (сворачивается кликом по шапке) + «Отозвать валидатор»/крестик.
  Крестик → `completeValidation` + финал.
- `final-accounts-screen.tsx` (шаг 29) — `AccountsScreen` + `useEnsureFinal`.

### [x] Фаза 3. Связи (accounts-screen)
Баннер processing «Перейти к документам» → чат. Новый баннер `validated` (синий
«Отправка на юрисдикцию»). Статус документа → «Отвалидирован» при validated. Клик
по документу в Документообороте (processing/validated) → чат. Свёрнутость карты
при позднем сиде (effect на configured). Маршруты 28/29.
