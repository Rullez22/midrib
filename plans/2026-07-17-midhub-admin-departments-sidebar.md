# Сайдбар: подразделения переезжают в Администрацию

**Задача.** Сейчас в кабинете Администрации после сепаратора — «Пайщики» и «Партнеры», а у
каждого кабинета 2–8 свои внутренние пункты (Заявки, Реестры, Цели…). Новая структура:
после сепаратора в Администрации — список всех 8 подразделений; клик по разделу открывает
кнопку «Назад» (к списку) и страницы этого подразделения. У самих подразделений внутренние
пункты уходят — в их сайдбаре остаётся только карточка-аватар + «Деятельность» и «Счета».

Пайщики/Партнеры становятся страницами пункта «Администрация».

## Что где показывается

| Пункт списка | После клика |
|---|---|
| Администрация | Назад · Пайщики · Партнеры |
| Валидатор | Назад · Заявки · Обработанные · Реферальные · Шаблоны |
| Веб-ресурс | Назад · Регистрации |
| Домены | Назад · Реестры |
| Исполнитель | Назад · Партнеры |
| Регулятор | Назад · Сервисы |
| ВУЗы | Назад · Направление · Выдача диплома · Дополнения · История операций |
| Фонд | Назад · Цели · Партнеры · Статистика |

ВУЗовские «Выдача диплома» и «Дополнения» по-прежнему скрыты до прохождения флоу
(`CabinetUnlock`, `lockedUntil`).

## Reuse-анализ (DS-first)

| Что | Откуда |
|---|---|
| Кнопка «Назад» | 1:1 из `partner-sidebar` (`BackIcon` + размер/стили кнопки) |
| Пункты меню | `MenuNavItem` (как сейчас Пайщики/Партнеры) |
| Сепаратор | `MenuDivider` |
| Цвет раздела | `BADGE_COLOR_VAR` по `railColorOf(slug)` — как у бейджа рейки |
| Пути страниц | `CABINET_LIST[].menu[].path` + `CABINET_ROUTES` для Администрации |

## Ключевые решения

- **Состояние раскрытия — в самом сайдбаре**, а не в URL: инициализируется из текущего пути
  (открыта страница подразделения → раздел раскрыт), «Назад» сбрасывает к списку. URL страниц
  не меняются — переезжает навигация, а не маршруты.
- **Онбординг не трогаем.** `CoopSidebar` живёт и в `/flow/company-create`, где кабинета ещё
  нет: там остаются Пайщики/Партнеры. Режим определяется по `usePathname()` (`/cabinet/*`).
- **Страницы-пункты кабинетов** (`/cabinet/validator/zayavki` и пр.) показывают сайдбар
  Администрации с раскрытым разделом — иначе после «переезда» пункт открывался бы в кабинете,
  где его уже нет в меню, и вернуться было бы некуда.

## Фазы

- [x] **1. Данные.** Список разделов для сайдбара: Администрация (Пайщики/Партнеры) + кабинеты
      2–8 из `CABINET_LIST`; пути, иконки, цвет рейки, `lockedUntil`.
- [x] **2. Блок «Подразделения».** Общий компонент: список 8 разделов ↔ раскрытый раздел
      (Назад + его пункты). Активный пункт подсвечен по текущему URL.
- [x] **3. Администрация.** `CoopSidebar` в кабинетном режиме: вместо Пайщики/Партнеры —
      новый блок. Онбординг — как было.
- [x] **4. Подразделения 2–8.** `CompanySidebar`: убрать сепаратор и `menu` — остаются
      карточка + Деятельность + Счета. На страницах-пунктах — сайдбар Администрации
      с раскрытым разделом.
- [x] **5. Проверка в браузере.** Все пункты всех разделов открываются и подсвечиваются;
      «Назад» возвращает к списку; у кабинетов 2–8 меню пустое; онбординг не задет; ВУЗ-замки
      работают.

## Итог

Реализован целиком.

Что получилось: в Администрации под сепаратором — 8 разделов; клик раскрывает «Назад» +
страницы этого подразделения (у Администрации это Пайщики/Партнеры). У кабинетов 2–8 своё
меню исчезло — остались карточка, «Деятельность», «Счета». URL страниц не менялись.

Отдельно: `CabinetUnlockProvider` пришлось поднять из `[company]/layout` в layout всего
`/cabinet` — пункты ВУЗа теперь открываются из Администрации, и снятая блокировка не должна
теряться при уходе из `/cabinet/vuz`.

Проверено в браузере: все 8 разделов дают ожидаемые пункты; переход по пункту открывает
страницу и подсвечивает её в раскрытом разделе; у кабинетов 2–8 nav-пунктов нет; онбординг
(`/flow/company-create/5`) сохранил Пайщики/Партнеры.
