## Аудит: ~51 нода = состояния одного конструктора

Все ноды — состояния/варианты конструктора шаблона, собранного из повторяющегося
каркаса + уже существующих DS-компонентов:

| Часть | Решение |
|---|---|
| Секция (шапка + тело + «+ Добавить …») — «Поля документа», «Требования», «Описание» | новый **`Panel`** |
| Сворачиваемые поля / требования (список) | **reuse `QuestionCard`** (size s) |
| Статусы требований (Жёлтый/Зелёный/Международный) | **reuse `Badge`** |
| Форма «Добавить поле» / описание | **reuse `PropertyForm` / `Textarea`** |
| «+ Добавить поле/требование/роль» | **reuse `Button`** (tertiary + plus) |

## Panel — новый каркас

Бордерная карточка: шапка (grey-10, заголовок) + тело (`children`, паддинг настраивается
`bodyClassName`) + опц. футер-кнопка «+ Добавить …» (по центру, tertiary, верхняя рамка).
Базовый каркас экранов-конструкторов. Токены MIDHUB, Tailwind-утилиты.
