## Аудит (5 нод) — почти полный reuse, 1 новый композит

Разбор формы на части:

| Часть | DS-компонент | reuse |
|---|---|---|
| Название роли (плавающий лейбл) | `Input` size=l | ✅ есть |
| Должностные обязанности (ввод) | `Textarea` size=l | ✅ есть |
| «Добавить требования» | `Button` variant=secondary | ✅ есть |
| Карточка требования (раскрытие) | `QuestionCard` | ✅ есть |
| Бейдж «Желтый/Международный» | `Badge` color=orange | ✅ есть |
| Документы (Паспорт/Партийный билет) | `Item` tone=muted | ✅ есть |
| Лейблы секций (16/24 Medium) | `ds-p2-medium` | ✅ токен |
| **Раскладка формы + read-only описание** | — | 🆕 `RoleForm` |

Новое — только сам **композит-обёртка `RoleForm`** (раскладка полей + режим
view с описанием read-only карточкой). Карточка требования не дублируется —
переиспользует паттерн [[2026-06-09-ds-requirement-template]]
(`QuestionCard`+`Badge`+`Item`).

## Фазы

### [x] Фаза 1. Композит RoleForm
- `src/components/ds/composite/role-form.tsx` — props: `mode` (create/view),
  `name`/`onNameChange`, `duties`/`onDutiesChange`, `children` (слот карточек
  требований), `onAddRequirement`.

  слот требований + Button «Добавить требования».
- view (готовая роль): описание — read-only карточкой (border + muted text),
  карточки требований, без Input/Button.
- Экспорт в `src/components/ds/index.ts`.
