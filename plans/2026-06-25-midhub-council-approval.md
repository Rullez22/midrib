# Согласование совета: добавление пайщиков (флоу «member»)

Источник: Figma 6537-355370 / 355569 / 355768 / 358122 / 356023 / 356212.
Экран: «Управление пайщиками кооператива» → таб **«Согласование совета»**.

## Суть

Пайщика выбирают в shuttle → «Отправить на совет» → он переходит в статус
«На согласовании» → вопрос «Добавление пайщиков» появляется в табе **Совет**
экрана голосования → после голосования пайщики появляются в **«Действующие
пайщики»**. Флоу аналогичен платежам, реализован переиспользованием.

## Фазы

- [x] **Фаза 1. Модель.** `PaymentVote.kind` расширен значением `"member"`.
  `finishPaymentVote` для `kind="member"` переносит `recipients` в
  `invitedMembers` (с дедупом). Файл: `reg-flow.tsx`.
- [x] **Фаза 2. Таб «Согласование совета».** В `paishiki-members-panel.tsx`:
  пока нет «pending» — `PaishikShuttle` (общий блок) + «Отправить на совет»
  (создаёт `member`-вопрос). После отправки — таблица «На согласовании» (розовая
  плашка-статус, колонки Адрес/Страна/Дата заявки). Shuttle обёрнут в
  `gap-4` (16px) как в каноне `payment-once`.
- [x] **Фаза 3. Вопрос в голосовании.** `paymentVotes` уже рендерятся только в
  табе `role==="sovet"` → «Добавление пайщиков» появляется там автоматически
  (иконка `share`). Кнопка-нав «Вопросы голосования» в сайдбаре загорается через
  существующий `paymentVoteActive`.
- [x] **Фаза 4. Детальный экран.** Новый `member-vote-detail.tsx` (по образцу
  `TokenVoteDetail`): «Основание для добавления в кооператив» (read-only) +
  «Выбранные пайщики» (Частные/Юридическое) + универсальный `VotingBlock`.
  Подключён в `voting-roles-screen.tsx` через `kind==="member"`.
- [x] **Фаза 5. Вход во флоу из формы.** Приглашение пайщика через форму в
  операционном кабинете (`PaishikiScreen inviteToCouncil`) добавляет его НЕ в
  «Действующие», а в кандидаты согласования (`RegFlow.councilCandidates`), и
  панель автопереключается на таб «Совет» (`focusCouncilSignal`). Shuttle берёт
  частных пайщиков из `councilCandidates` (параметр `privateNames` у
  `PaishikShuttle`); отправленные на совет удаляются из кандидатов
  (`removeCouncilCandidates`) и уходят в «На согласовании». Статус-плашка «На
  согласовании» приведена к Figma (`rgba(229,66,77,0.3)` / `#e5424d`, 12/20).

## Переиспользование (DS / композиты)

`PaishikShuttle`, `PRIVATE/LEGAL/ADDR` (payment-shared) · `VotingBlock`,
`InfoIcon` (voting-block) · `Tabs/Tab`, `Button`, `Toolbar`, `SearchBar`,
`Dropdown`, `Link`, `EmptyState`, `SectionHeader`, `HeaderArrowLeftIcon`,
`CoopRail` (DS) · паттерн detail-страницы из `PaymentVoteDetail/TokenVoteDetail`.
Новой вёрстки с нуля — минимум (только статичные read-only таблицы основания и
выбранных пайщиков).

## Итог

Реализован целиком. Полный цикл проверен в браузере (Playwright): shuttle →
отправка → «На согласовании» → вопрос в табе Совет → детальный экран
«Добавление пайщиков» → после голосования пайщики в «Действующие пайщики»
(новые добавлены, дубли отброшены).
